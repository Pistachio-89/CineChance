# План реализации страниц детализации статистики (Оценки / Теги / Жанры)

## Цель
Добавить отдельные страницы для отображения списков фильмов по выбранному элементу статистики:
- Оценки (например, 10)
- Теги (например, "Ностальгия")
- Жанры (например, "Боевик")

Страница должна показывать карточки фильмов со всем существующим функционалом. UX/UI не менять.

---

## 1. Анализ текущей структуры
1.1. Определить, где формируются статистические блоки Оценки/Теги/Жанры (страница профиля/дашборда, компонент статистики).
1.2. Найти используемую карточку фильма и связанные компоненты (например, `MovieCard`, списки фильмов, хук/контекст для взаимодействий).
1.3. Найти существующие маршруты (pages/app router) и как формируются списки фильмов (есть ли страница списка по фильтрам).

Результат: список файлов и компонентов, которые будут переиспользованы без изменений UX/UI.

---

## 2. Проектирование маршрутов
2.1. Определить схему URL для каждой категории:
- `/stats/ratings/[rating]`
- `/stats/tags/[tag]`
- `/stats/genres/[genre]`

2.2. Убедиться, что это не конфликтует с существующими маршрутами.

---

## 3. Добавление страниц
3.1. Создать три страницы (App Router или Pages Router в зависимости от текущей архитектуры):
- Страница `ratings`: принимает параметр `rating` и отображает список фильмов.
- Страница `tags`: принимает параметр `tag`.
- Страница `genres`: принимает параметр `genre`.

3.2. Каждая страница:
- Получает список фильмов для текущего пользователя.
- Отображает карточки фильмов (существующий компонент).
- Соблюдает текущие стили/сетки/адаптивность.

---

## 4. Оптимизация данных
4.1. Не перегружать запросы: использовать селективные выборки и/или переиспользование уже существующих API/функций.
4.2. Проверить, есть ли у API/DAO методы фильтрации по оценке/тегу/жанру.
4.3. При необходимости добавить новые методы в слой данных:
- `getMoviesByRating(userId, rating)`
- `getMoviesByTag(userId, tag)`
- `getMoviesByGenre(userId, genre)`

---

## 5. Связь со статистикой
5.1. В блоках статистики добавить кликабельность:
- По лику на бар/лейбл/элемент — переход на страницу детального списка.
5.2. UI не менять: использовать существующий стиль (например, сделать элемент ссылкой/кнопкой с тем же внешним видом).

---

## 6. Поведение карточки фильма
6.1. Убедиться, что карточка фильма работает как в остальных списках:
- Лайки/избранное
- Рейтинг/оценки
- Взаимодействия (если есть)

---

## 7. Адаптивность и UX
7.1. Проверить адаптивность на стандартных брейкпоинтах.
7.2. Не добавлять новые UI элементы, которые изменяют восприятие страницы.

---

## 8. Проверка и тестирование
8.1. Проверить переходы по клику.
8.2. Проверить корректность фильтрации.
8.3. Проверить производительность (нет лишних запросов).

---

## 9. Документация
9.1. Кратко описать новые маршруты и фильтры в существующей документации проекта (если есть).

---

## Итог
Пользователь получает отдельные страницы с детальными списками фильмов по оценке, тегу и жанру с полным функционалом карточек, при неизменном UX/UI и оптимизированных запросах.
